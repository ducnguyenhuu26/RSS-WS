# DUC-WM: LLM-Guided Mechanism-Residual World Model

DUC-WM is a model-based control framework for continuous-control environments
whose transition dynamics depend on hidden but inferable mechanisms. The method
does not ask an LLM to predict dynamics directly. Instead, the LLM defines a
safe mechanism-law hypothesis space: mechanism names, masks, confidence, and a
small DSL law type such as velocity damping, action delay, or external drift.
Data then decides which mechanisms are active, how strongly they act, and how
much each compiled prior law must be repaired by a mechanism-specific neural
residual.

The central claim is conditional:

> If deployment dynamics are generated by reusable latent mechanisms, then a
> mechanism-residual world model can improve compositional OOD prediction,
> attribution, and model-based planning relative to context-free or black-box
> latent-context world models.

DUC-WM is designed for hidden mechanisms such as wind, friction shift, damping
shift, mass shift, action delay, sticky transition, impulse force, and gravity
shift.

## Problem

Consider a family of continuous-control environments:

$$
e \sim p(e), \qquad x_t \in \mathbb R^d, \qquad a_t \in \mathbb R^m.
$$

The true transition depends on an unobserved context:

$$
x_{t+1}=F^\star(x_t,a_t,c_e,c_t^{event})+\epsilon_t.
$$

Here \(c_e\) describes persistent environment physics, while
\(c_t^{event}\) describes transient events. The agent observes only history:

$$
h_t=(x_{t-L},a_{t-L},\dots,x_t,a_t).
$$

The control objective is:

$$
\max_\pi \mathbb E_e[J_e(\pi)],
\qquad
J_e(\pi)=
\mathbb E\left[\sum_{t=0}^{T}\gamma^t r_e(x_t,a_t)\right].
$$

The modeling objective is not only to predict \(x_{t+1}\). DUC-WM also tries to
explain the transition through named, reusable mechanism contributions.

## Final Model

DUC-WM models the state delta:

$$
\Delta x_t=x_{t+1}-x_t
$$

as an invariant base transition plus sparse hidden-mechanism shifts:

$$
\hat x_{t+1}
=
x_t+B_\theta(x_t,a_t)
+
\sum_{j=1}^{K}
\alpha_{j,t}
\left[
\beta_j P_j^{LLM}(x_t,a_t,h_t)
+
g_t R_{\theta,j}(x_t[P_j^x],a_t[P_j^a])
\right].
$$

The base trunk \(B_\theta\) learns ordinary MuJoCo dynamics shared across contexts.
The compiled law prior \(P_j^{LLM}\) is selected from a safe DSL emitted by the
LLM or fallback template bank. The learned scalar \(\beta_j\) calibrates law
strength from data, while \(g_t \in [0,1]\) opens the neural residual gradually
during training. The residual bank \(R_{\theta,j}\) learns reusable corrections
caused by hidden mechanisms such as wind, friction, mass, damping, delay,
sticky transitions, impulse, and gravity.

The components are:

| Symbol | Meaning |
|---|---|
| \(B_\theta\) | shared base dynamics trunk |
| \(P_j^{LLM}\) | compiled safe law prior for mechanism \(j\) |
| \(\beta_j\) | learned positive calibration of the law prior |
| \(g_t\) | residual warmup gate during training |
| \(R_{\theta,j}\) | mechanism-specific neural residual |
| \(\alpha_{j,t}\) | bounded inferred strength of mechanism \(j\) |
| unknown slot | dense fallback mechanism for missing or wrong priors |

The current implementation uses safe executable law-DSL priors. The LLM/fallback
template specifies mechanism names, masks, `law_type`, `law_gain`, scales,
confidence, timescale, and reward relevance. Code then compiles each DSL law into
a tensor prior effect; trainable residual networks learn only the missing or
wrong part.

DUC-WM applies a confidence-weighted trust region around each law prior:

$$
\mathcal L_{TR}
=
\sum_j \rho_j
\left[
\|g_t R_{\theta,j}\|_2
-
\delta_j \|\beta_j P_j^{LLM}\|_2
\right]_+^2,
\qquad
\delta_j=\delta_{min}+(1-\rho_j)\delta_{range}.
$$

High-confidence priors permit only small neural corrections, while
low-confidence priors leave more room for data-driven repair. This prevents
DUC-WM from collapsing back into an MLP-style residual model.

## Mechanism Templates

Each mechanism template is:

$$
T_j=
(\mathrm{name}_j,P_j^x,P_j^a,O_j,\ell_j,g_j,s_j,
\mu_j,\sigma_j,\rho_j,\tau_j,\eta_j).
$$

| Field | Meaning |
|---|---|
| \(\mathrm{name}_j\) | mechanism name |
| \(P_j^x\) | state dimensions read by the mechanism |
| \(P_j^a\) | action dimensions read by the mechanism |
| \(O_j\) | state dimensions written by the mechanism |
| \(\ell_j\) | safe DSL law type, e.g. velocity_damping or action_delay |
| \(g_j\) | small prior law gain |
| \(s_j\) | maximum strength scale |
| \(\mu_j,\sigma_j\) | prior mean and uncertainty for raw context |
| \(\rho_j\) | prior confidence |
| \(\tau_j\) | timescale: slow, event, or unknown |
| \(\eta_j\) | reward relevance explanation |

The LLM is called offline. It receives the environment name, state/action index
contract, task objective, reward-critical dimensions, allowed mechanism
families, and a strict JSON schema. The output JSON can be reused across seeds.

## Slow and Event Context

DUC-WM separates persistent physics from transient events:

$$
q_\phi(c_t \mid h_t)
=
q_\phi(c_e^{slow},c_t^{event}\mid h_t).
$$

Slow mechanisms use coarse history statistics, such as average state/action and
history-level drift. Examples:

$$
\{\text{wind},\text{friction},\text{mass},\text{damping},\text{gravity}\}.
$$

Event mechanisms use the recent window directly. Examples:

$$
\{\text{delay},\text{sticky},\text{impulse},\text{unknown}\}.
$$

The bounded strength is:

$$
\alpha_{j,t}=s_j\tanh(u_{j,t}),
\qquad
| \alpha_{j,t} | \le s_j.
$$

This prevents rollout explosions and keeps mechanism strength interpretable.

## Probabilistic Transition

The conditional transition model is:

$$
p_\theta(x_{t+1}\mid x_t,a_t,h_t)
=
\mathcal N(\mu_\theta,\Sigma_\theta),
$$

where:

$$
\mu_\theta=
x_t+B_\theta(x_t,a_t)+\sum_j \alpha_{j,t}G_{\theta,j}(x_t[P_j^x],a_t[P_j^a]).
$$

The covariance is diagonal:

$$
\Sigma_\theta=\operatorname{diag}(\exp(\ell_\theta(x_t,a_t,\alpha_t))).
$$

## Training Modes

DUC-WM supports two modes.

### Supervised Virtual Context

This mode is for debugging and attribution validation when synthetic context
labels are available:

$$
\mathcal L
=
\mathcal L_{nll}
+
\lambda_{ctx}\|\hat c-c^\star\|_2^2
+
\lambda_{ctrl}\mathcal L_{ctrl}.
$$

The model can optionally receive the true context during training through
`teacher_force_context=true`. This should be treated as a debugging mode, not
the main deployment claim.

### Latent Context Inference

This is the main scientific setting:

$$
\mathcal L
=
\mathcal L_{nll}
+
\beta\operatorname{KL}(q_\phi(c\mid h)\|p_{LLM}(c))
+
\lambda_{res}\mathcal L_{res}
+
\lambda_{orth}\mathcal L_{orth}
+
\lambda_{sparse}\|\alpha\|_1
+
\lambda_{unk}\mathcal L_{unk}
+
\lambda_{roll}\mathcal L_{roll}
+
\lambda_{ctrl}\mathcal L_{ctrl}.
$$

The default `configs/config.yaml` uses latent inference:

```yaml
duc:
  context_weight: 0.0
  teacher_force_context: false
```

## Loss Terms

Negative log likelihood:

$$
\mathcal L_{nll}
=
-\mathbb E[\log p_\theta(x'\mid x,a,h)].
$$

KL to the LLM/context prior:

$$
\mathcal L_{KL}
=
\operatorname{KL}(q_\phi(c\mid h)\|p_{LLM}(c)).
$$

Confidence-weighted residual repair:

$$
\mathcal L_{res}
=
\sum_{j=1}^K
\rho_j
\|R_{\theta,j}(x,a,h)\|_2^2.
$$

High-confidence priors receive stronger residual regularization. Low-confidence
priors are easier for data to repair.

Unknown mechanism penalty:

$$
\mathcal L_{unk}=|\alpha_{0,t}|.
$$

If the unknown slot activates often, the prior is missing an important
mechanism.

Control-weighted prediction:

$$
\mathcal L_{ctrl}
=
\|W_{ctrl}^{1/2}(\hat x'-x')\|_2^2.
$$

Rollout consistency:

$$
\mathcal L_{roll}
=
\frac{1}{H}
\sum_{k=0}^{H-1}
\|W_{ctrl}^{1/2}(\hat x_{t+k+1}-x_{t+k+1})\|_2^2.
$$

Mechanism separation:

$$
\mathcal L_{orth}
=
\sum_{i\ne j}
\langle W_{ctrl}^{1/2}G_i,W_{ctrl}^{1/2}G_j\rangle^2.
$$

Sparse activation:

$$
\mathcal L_{sparse}=\|\alpha_t\|_1.
$$

## MuJoCo Extension

The extension collector creates virtual hidden-mechanism environments. It saves:

$$
\mathcal D=
\{x_t,a_t,x_{t+1},r_t,d_t,c_t\}_{t=1}^N.
$$

Context mechanisms include:

| Mechanism | Type | Implementation role |
|---|---|---|
| wind | slow | persistent velocity drift |
| friction | slow | contact/velocity attenuation |
| mass | slow | body mass parameter shift |
| damping | slow | damping parameter shift |
| gravity | slow | gravity parameter shift |
| delay | event | mixes current and previous actions |
| sticky | event | sticky action or reduced transition delta |
| impulse | event | rare velocity perturbation |

The default template bank also includes an `unknown` mechanism that is not
present in the synthetic labels and is regularized toward inactivity.

## Evaluation

A defensible DUC-WM result should report prediction, attribution, rollout, and
control metrics:

| Metric | Purpose |
|---|---|
| \(R^2@1\) | one-step prediction |
| \(R^2@H\) | multi-step rollout prediction |
| \(R^2_{DUC}@1\) | control-weighted one-step prediction |
| \(R^2_{DUC}@H\) | control-weighted rollout prediction |
| context MSE | strength inference quality when labels exist |
| attribution accuracy | whether the active mechanism is correctly identified |
| unknown activation | whether the named prior is sufficient |
| MPC return | final model-based control performance |

Ordinary \(R^2\):

$$
R^2
=
1-
\frac{\sum_t\|x_t-\hat x_t\|_2^2}
{\sum_t\|x_t-\bar x\|_2^2}.
$$

Control-weighted \(R^2\):

$$
R^2_{DUC}
=
1-
\frac{\sum_t\|x_t-\hat x_t\|_{W_t}^2}
{\sum_t\|x_t-\bar x\|_{W_t}^2}.
$$

Attribution accuracy uses the known synthetic intervention label:

$$
\arg\max_j |c_j^\star|
\quad \text{vs.} \quad
\arg\max_j |\hat \alpha_j|.
$$

## Compositional OOD Benchmark

The main benchmark should test whether mechanisms recombine compositionally.

Train on single or limited mechanisms:

$$
\{\text{wind}\},\{\text{friction}\},\{\text{mass}\},\{\text{delay}\}.
$$

Test on unseen combinations:

$$
\{\text{wind}+\text{friction}\},
\quad
\{\text{mass}+\text{delay}\},
\quad
\{\text{gravity}+\text{sticky}\}.
$$

This is the most important evidence that DUC-WM is more than a generic
latent-context model.

## Planning

DUC-WM supports MPC-style planning:

$$
a^\star_{t:t+H}
=
\arg\max_{a_{t:t+H}}
\mathcal J_\theta.
$$

The model objective is:

$$
\mathcal J_\theta
=
\mathbb E_{c\sim q_\phi(c\mid h_t)}
\left[
\sum_{\ell=0}^{H}\gamma^\ell r(\hat x_{t+\ell},a_{t+\ell})
\right]
-
\beta_U U.
$$

Uncertainty combines model variance and context variance:

$$
U=U_{model}+U_{ctx}.
$$

## Required Ablations

The ablation plan is part of the method:

| Ablation | Question |
|---|---|
| black-box MLP world model | is mechanism structure needed? |
| black-box latent-context model | are named strengths needed? |
| DUC without LLM prior | does structure help? |
| random prior | is any decomposition enough? |
| wrong prior | can data repair bad priors? |
| no unknown slot | does the model over-attribute missing mechanisms? |
| no regularization | do residual/unknown/sparse/orth terms matter? |
| no rollout loss | does rollout training improve multi-step prediction and MPC? |

The strongest positive result is:

1. LLM prior beats random and wrong priors.
2. Mechanism-specific residual beats global residual.
3. Unknown activation rises when the prior omits a true mechanism.
4. Attribution tracks the true intervention.
5. Control-weighted rollout error correlates with MPC return.

## Run

Install/sync dependencies if needed:

```bash
uv sync --dev
```

Smoke run:

```bash
uv run python main.py --config-name smoke
```

Default latent-context run:

```bash
uv run python main.py problem=Hopper-v5 device=cuda
```

Workshop sweep:

```bash
uv run python main.py --config-name workshop -m \
  model=mlp,pets,cadm,duc_no_llm,duc_random,duc_wrong_prior,duc_no_unknown,duc_no_reg,duc_no_rollout,duc_wm \
  problem=Hopper-v5,Walker2d-v5 \
  seed=0,1,2 \
  device=cuda
```

Large-machine sweep with CPU-parallel collection and BF16 training:

```bash
uv run python main.py --config-name workshop_big -m \
  model=mlp,pets,cadm,duc_no_llm,duc_random,duc_wrong_prior,duc_no_unknown,duc_wm \
  problem=Hopper-v5,Walker2d-v5 \
  seed=0,1,2 \
  mujoco_extension.parallel_workers=0 \
  runtime.precision=bf16 \
  runtime.preload_to_device=true \
  device=cuda
```

`parallel_workers=0` uses all logical CPU cores for MuJoCo collection. On a
shared machine, set it to a fixed value such as `16` or `32`.

Balanced CPU/GPU pilot on one rented GPU:

```bash
uv run python scripts/run_balanced_gpu_jobs.py \
  --config-name pilot_swimmer_gpu \
  --models mlp,pets,cadm,duc_wm \
  --max-parallel 2
```

This launcher starts one process per method. Keep `--max-parallel=2` on a
single GPU unless monitoring shows plenty of free VRAM and low utilization.
Each process uses CPU workers for MuJoCo collection/env stepping and CUDA for
training, rollout evaluation, and CEM candidate scoring.

Compositional OOD sweep:

```bash
uv run python main.py --config-name workshop -m \
  model=mlp,pets,cadm,duc_no_llm,duc_random,duc_wrong_prior,duc_wm \
  'mujoco_extension.train_variants=[wind,friction,delay,mass]' \
  'mujoco_extension.test_variants=[wind+friction,friction+delay,mass+delay]' \
  seed=0,1,2 \
  device=cuda
```

The implemented methods are:

| Method | Role |
|---|---|
| `mlp` | black-box Gaussian MLP dynamics |
| `pets` | PETS-style probabilistic ensemble dynamics |
| `cadm` | CaDM-style latent-context dynamics |
| `duc_no_llm` | DUC architecture with non-semantic generic mechanisms |
| `duc_random` | DUC architecture with random mechanism masks |
| `duc_wrong_prior` | DUC architecture with rotated/wrong semantic masks |
| `duc_no_unknown` | DUC-WM without the unknown fallback mechanism |
| `duc_no_reg` | DUC-WM without residual/unknown/sparse/orth regularizers |
| `duc_no_rollout` | DUC-WM without rollout-consistency training |
| `duc_wm` | full DUC-WM with LLM/fallback law-DSL priors plus neural residuals |

Planning is controlled by the `planning` block. `workshop.yaml` and
`workshop_big.yaml` enable CEM-MPC by default, so output JSON files include
`score.planner_return_mean` and `score.planner_return_std`. `smoke.yaml` keeps
planning disabled unless explicitly overridden.

## Current Implementation Status

Implemented:

- LLM/fallback safe mechanism law-DSL templates;
- slow/event/unknown template metadata;
- compiled tensor law priors plus mechanism-specific residual shifts;
- learned prior calibration \(\beta_j\);
- residual warmup gate and confidence-weighted trust-region loss;
- invariant base dynamics trunk plus mechanism-specific shifts;
- split context encoder for slow and event mechanisms;
- bounded mechanism strengths;
- unknown residual slot in the default template bank;
- confidence-weighted residual regularization on neural corrections only;
- unknown activation regularization;
- rollout and attribution metrics;
- CEM-MPC reward evaluation through a learned reward surrogate;
- latent-context default config and supervised smoke config.
- MLP, PETS-style, CaDM-style, DUC ablation, and full DUC-WM benchmark methods;
- workshop config and grouped result aggregation script.

Still recommended before making a strong paper claim:

- run compositional OOD train/test splits at scale;
- validate CEM-MPC returns against stronger model-based planning baselines;
- report robustness to random, wrong, and missing LLM priors.
